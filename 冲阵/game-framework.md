# 竖版塔防对冲战斗 - 游戏框架设计文档

## 1. 核心战斗系统

### 1.1 战场布局
- **方向**：竖版，左中右三线（横向布局）
- **出兵方向**：我方从下方出兵，敌方从上方出兵，向中线对冲
- **胜利条件**：摧毁敌方基地 / 敌方基地血量归零
- **对局时长**：约1分钟

### 1.2 三线差异设计
| 路线 | 特征 | 战术意义 |
|------|------|----------|
| 左路 | 距离较长，移动速率-20% | 慢速推进，适合远程消耗 |
| 中路 | 标准距离，标准速率 | 主战场，兵力集中 |
| 右路 | 距离较短，移动速率+20% | 快速接敌，适合突袭 |

### 1.3 战斗机制
- **碰撞**：阻挡机制，单位有碰撞体积，可形成战线
- **战斗模式**：实时自动战斗
- **攻击类型**：
  - 近战：接触后进入攻击间隔
  - 远程：进入射程后开始攻击
- **死亡**：血条归零后消失，可能触发亡语效果

### 1.4 英雄系统
- **普攻**：自动进行
- **大招**：手动释放，单独CD 20秒
- **数量**：单局1-3个（局外养成带入）

## 2. 地形系统

### 2.1 空间层
```
┌─────────────────────────┐
│      【空中层】          │  ← 空中单位通行，可攻击地面
├─────────────────────────┤
│      【地面层】          │  ← 主战场（陆地/水面/桥梁/高地）
├─────────────────────────┤
│      【地下层】          │  ← 预留扩展
└─────────────────────────┘
```

### 2.2 地形类型
| 地形 | 通行规则 | 特殊效果 |
|------|----------|----------|
| 陆地 | 地面单位正常通行 | 无 |
| 水面 | 地面单位不可通行 | 需空中单位或桥梁绕过 |
| 桥梁 | 狭窄通道 | 易形成choke point |
| 高地 | 单向通行或特定单位 | 高地打低地有射程/伤害加成 |

### 2.3 空间层互动
- **空中 → 地面**：可攻击
- **地面 → 空中**：部分单位可攻击（弓箭手、法师等）
- **空中 vs 空中**：独立空战

### 2.4 埋伏扩展（后期）
| 埋伏类型 | 机制 |
|----------|------|
| 侧翼伏击 | 从边路高地出现，攻击侧翼 |
| 水下潜伏 | 水面下隐藏，接近后上浮 |
| 空降突袭 | 空中层切入，直扑后排 |

## 3. 单位系统

### 3.1 单位分类

#### 小兵
| 类型 | 定位 | 特性 |
|------|------|------|
| 近战 | 前排 | 高血量，阻挡敌方 |
| 远程 | 输出 | 后排输出，脆皮 |
| 坦克 | 肉盾 | 极高血量，低伤害，强阻挡 |

- **获取**：局外养成解锁/升级
- **局内**：可升级属性，数量由经济决定

#### 英雄
| 维度 | 设计空间 |
|------|----------|
| 定位 | 输出 / 坦克 / 辅助 / 控制 |
| 大招类型 | AOE伤害 / 召唤 / 治疗 / 位移 / 地形改变 |
| 被动特性 | 同类型小兵加成 / 击杀回复 / 复活等 |
| 空间适应 | 地面 / 飞行 / 两栖 |

#### 埋伏（后期扩展）
- 利用地形/空间层进行非正面攻击
- 有准备时间或触发条件

### 3.2 克制关系

#### 空间克制
- 能打到即优势
- 水面为独立类别

#### 兵种克制（保留设计空间）
- 经典三角：近战 > 远程 > 坦克 > 近战（示例）
- 通过兵种克制实现类似空间压制效果

## 4. 经济/建造系统

### 4.1 资源机制
- **基础增长**：自动随时间增加
- **击杀奖励**：击杀敌方单位获得额外资源
- **曲线**：0:00(1x) → 0:15(1.2x) → 0:30(1.5x) → 0:45(2x)

### 4.2 出兵机制
- **预带兵力**：开局前配置初始兵力
- **预设队列**：设置出兵顺序
- **兵种切换**：可随时切换，但有CD
  - 基础CD：3秒
  - 高级兵种：CD更长（按兵种等级递增）
- **升级**：独立按钮，暂停当前lane出兵进行升级

### 4.3 英雄与经济
- 英雄不消耗资源，CD制
- 部分英雄可能有经济相关被动

## 6. 技术架构

### 6.1 技术选型

| 层面 | 选择 | 说明 |
|------|------|------|
| **平台** | H5全平台 | 浏览器即可运行，便于分享和测试 |
| **引擎** | 原生Canvas 2D | 轻量、无依赖、完全可控 |
| **架构** | 单机纯客户端 | Demo阶段，无服务端 |
| **模块** | ECS-like | 实体-组件-系统模式，便于扩展 |

### 6.2 项目结构

```
冲阵/
├── index.html              # 入口
├── css/
│   └── style.css
├── js/
│   ├── core/               # 核心引擎
│   │   ├── Game.js         # 游戏主循环
│   │   ├── Entity.js       # 实体基类
│   │   ├── Component.js    # 组件基类
│   │   └── System.js       # 系统基类
│   ├── systems/            # 游戏系统
│   │   ├── BattleSystem.js # 战斗管理
│   │   ├── UnitSystem.js   # 单位管理
│   │   ├── MoveSystem.js   # 移动系统
│   │   ├── CollisionSystem.js # 碰撞检测
│   │   └── RenderSystem.js # 渲染系统
│   ├── components/         # 组件定义
│   │   ├── Transform.js    # 位置/旋转/缩放
│   │   ├── Health.js       # 生命值
│   │   ├── Movement.js     # 移动属性
│   │   ├── Combat.js       # 战斗属性
│   │   └── Renderer.js     # 渲染属性
│   ├── entities/           # 实体工厂
│   │   ├── Unit.js         # 单位创建
│   │   ├── Hero.js         # 英雄创建
│   │   └── Base.js         # 基地创建
│   ├── config/             # 配置数据
│   │   └── units.js        # 单位配置表
│   └── utils/              # 工具函数
│       ├── Vector2.js      # 2D向量
│       └── Pool.js         # 对象池
└── assets/                 # 资源文件（预留）
```

### 6.3 核心循环

```
Game Loop (60fps)
├── Update Phase
│   ├── BattleSystem.update()   # 战斗状态检查
│   ├── UnitSystem.update()     # 单位生成/销毁
│   ├── MoveSystem.update()     # 位置更新
│   └── CollisionSystem.update() # 碰撞检测
├── Render Phase
│   └── RenderSystem.render()   # 绘制所有实体
└── Input Phase
    └── 处理用户输入（兵种切换、英雄大招）
```

### 6.4 关键技术点

| 功能 | 实现方案 |
|------|----------|
| **碰撞检测** | AABB（轴对齐包围盒），分lane检测 |
| **对象池** | 预创建单位对象，避免GC卡顿 |
| **渲染优化** | 脏矩形重绘 / 分层渲染（后期） |
| **配置驱动** | JSON配置表定义单位属性 |
